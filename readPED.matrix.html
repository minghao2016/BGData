Creates an BGData object from a raw PED file (generated with --recodeA in PLINK) or a PED-like file and stores the genotypes in an in-memory matrix.

Usage

readPED.matrix(fileIn, header, dataType, n = NULL, p = NULL, na.strings = "NA", nColSkip = 6, idCol = c(1, 2), verbose = FALSE)

Arguments

fileIn
The path to the plaintext file.
header
If TRUE, the file contains a header.
dataType
The coding of genotypes. Use character() for A/C/G/T or integer() for numeric coding.
n
The number of individuals.
p
The number of markers.
na.strings
The character string used in the plaintext file to denote missing value.
nColSkip
The number of columns to be skipped to reach the genotype information in the file.
idCol
The index of the ID column. If more than one index is given, both columns will be concatenated with "_".
verbose
If TRUE, progress updates will be posted.

Value

Returns a BGData object.

Description

readPED.matrix assumes that the plaintext file (fileIn) contains records of individuals in rows, and phenotypes, covariates and markers in columns. The columns included in columns seq_len(nColSkip) are used to populate the slot @pheno of a BGData object, and the remaining columns are used to fill the slot @geno. If the first row contains a header (header=TRUE), data in this row is used to determine variables names for @pheno and marker names for @map and @geno.

Details

Genotypes are stored in a regular matrix object. Therefore, this function will only work if the raw PED file is small enough to fit into memory.
