Reads chunks of data from a memory-mapped file into memory and applies a function on each row or column of a matrix in parallel.

Usage

chunkedApply(X, MARGIN, FUN, bufferSize, i = seq_len(nrow(X)), j = seq_len(ncol(X)), nTasks = parallel::detectCores(), mc.cores = parallel::detectCores(), verbose = FALSE, ...)

Arguments

X
A matrix-like object, typically @geno of a BGData object.
MARGIN
The subscripts which the function will be applied over. 1 indicates rows, 2 indicates columns.
FUN
The function to be applied.
bufferSize
The number of rows or columns of X that are brought into memory for processing.
i
(integer, boolean or character) Indicates which rows should be used. By default, all rows are used.
j
(integer, boolean or character) Indicates which columns should be used. By default, all columns are used.
nTasks
The number of submatrices of each buffered subset of X to be processed in parallel.
mc.cores
The number of cores (passed to mclapply).
verbose
Whether to print additional information.
...
Additional arguments to be passed to parallelApply.

Description

bufferSize and nTasks have to be chosen carefully to avoid running out of memory. As a rule of thumb, at least around object_size(buffer) + (mc.cores * (object_size(buffer) / nTasks)) + object_size(result) MB of total memory will be needed, not including potential copies of your data that might be created (for example lsfit runs cbind(1, X)). Therefore, for 20 nodes and 20 tasks you will need at least 2 * object_size(buffer) MB, for 20 nodes and 40 tasks 1.5 * object_size(buffer) MB, etc.

Details

This function is only useful for memory-mapped files. For data that is already in memory, use parallelApply directly.
